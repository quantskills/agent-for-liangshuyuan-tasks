# 自动止盈止损+仓位管理（B11）

## 任务背景

在量化交易中，持仓管理是风控体系的核心环节。传统的止盈止损依赖人工盯盘，响应速度慢且容易因情绪波动错失最佳平仓时机。同时，单票仓位过重是交易账户爆仓的常见原因——没有仓位上限约束的止盈止损策略，在面对黑天鹅事件时可能造成远超预期的损失。

B11 的定位是为 A 股和期货交易提供一套轻量级的自动化持仓管理规则引擎：每日开盘后根据入场日期和开盘价，自动判断是否需要止盈、止损、强平或仓位调整，输出标准化平仓指令。与 B12 的盘中实时动态仓位管理互补，B11 侧重于入场后的离场纪律执行。

## 核心改动

**四级优先级的离场规则**：止盈和止损优先级最高——入场次日高开 ≥ 5% 触发止盈，低开 ≤ -3% 触发止损。若次日未触发止盈止损，且持仓天数 ≥ 2 日（含入场日），则强制平仓兜底。这一设计确保了"有机会止盈先止盈，没机会也不能无限期持仓"。

**单票仓位动态检查**：优先级最低但同样关键——单票持仓市值超过总资金 10% 时自动减仓至 10% 仓位线。数量操作向下取整到品种最小交易单位（A 股 100 股、期货 1 手）。这层约束防止单一品种的风险过度集中。

**A 股与期货共用规则**：B11 对 A 股和期货使用同一套决策逻辑，仅在最小交易单位上做区分。代码通过识别规则（6 位数字 = A 股，IF/IC/IH/IM 前缀 = 股指期货，小写字母前缀 = 商品期货）自动判断品种类别并应用正确的取整单位。

**纯 Python 零依赖**：整个工具仅使用标准库（datetime、re、json），无 pandas/numpy 等三方依赖，任意 Python 3.8+ 环境可直接运行。

## 测试情况

B11 交付了完整的自测脚本，覆盖 16 个测试用例：输入校验（通过/缺字段/价格异常/日期异常）、次日止盈、次日止损、持仓 2 日强平、止盈优先于强平、入场当天 hold、单票超 10% 减仓（A 股和期货）、批量调用、边界值（恰好 +5% 和 -3%）、空列表。全部 16 项通过。

## 总结与展望

B11 提供了一套简洁但完整的入场后持仓管理方案，适合作为交易 Agent 或 Alpha 信号的离场执行组件。后续可扩展方向包括：接入交易日日历以区分自然日与交易日、支持空头/双向持仓、接入涨跌停/停牌状态以提前判断流动性风险。
